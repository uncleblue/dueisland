MCP Setup Guide

DueIsland · Connect any AI client to your reminders

DueIsland ships with a built-in local MCP server, so any MCP-capable AI client (Claude Code, Claude Desktop, Cursor, Codex, VS Code, and more) can create, list, update, complete, snooze, and delete your reminders. Changes show up instantly in the notch and the main window.

Enable

  1. Open DueIsland ▸ Settings ▸ MCP Server and turn on the switch.
  2. Copy the URL and the auth header shown in Settings (unique per device; resetting the token invalidates old configs).

The server only listens on 127.0.0.1 (your Mac, plain HTTP — loopback traffic never leaves the machine and public CAs don't issue certificates for localhost, so HTTPS is neither possible nor needed here) and requires Authorization: Bearer <token>.

Universal connection info

Every client needs just these two values:

FieldValue
URLhttp://127.0.0.1:<PORT>/mcp
HeaderAuthorization: Bearer <TOKEN>
TransportStreamable HTTP

<PORT> defaults to 51837; if occupied, DueIsland automatically moves to the next free port — always use the URL shown in Settings. Copy <TOKEN> from Settings. Prefer 127.0.0.1 over localhost: some clients resolve localhost to IPv6 ::1, which the server does not bind.

Client configuration

Claude Code

claude mcp add --transport http dueisland http://127.0.0.1:<PORT>/mcp \
  --header "Authorization: Bearer <TOKEN>"

Verify with claude mcp list or /mcp inside a session.

Cursor

~/.cursor/mcp.json (global) or .cursor/mcp.json in a project:

{
  "mcpServers": {
    "dueisland": {
      "url": "http://127.0.0.1:<PORT>/mcp",
      "headers": { "Authorization": "Bearer <TOKEN>" }
    }
  }
}

Claude Desktop

claude_desktop_config.json (Settings ▸ Developer ▸ Edit Config):

{
  "mcpServers": {
    "dueisland": {
      "url": "http://127.0.0.1:<PORT>/mcp",
      "headers": { "Authorization": "Bearer <TOKEN>" }
    }
  }
}

OpenAI Codex

Desktop app: fill in the “Headers” field directly — no environment variables needed:

  • URL: http://127.0.0.1:<PORT>/mcp
  • Header key Authorization, value Bearer <TOKEN>
  • Leave “Bearer token environment variable” empty

CLI / config.toml: Codex references the token via an environment variable name:

export DUEISLAND_TOKEN="<TOKEN>"
[mcp_servers.dueisland]
url = "http://127.0.0.1:<PORT>/mcp"
bearer_token_env_var = "DUEISLAND_TOKEN"

Note: desktop GUI apps usually can’t read variables exported in your shell — prefer the Headers method in the GUI.

VS Code (GitHub Copilot, Continue, …)

.vscode/mcp.json (note the outer key is servers and type is required):

{
  "servers": {
    "dueisland": {
      "type": "http",
      "url": "http://127.0.0.1:<PORT>/mcp",
      "headers": { "Authorization": "Bearer <TOKEN>" }
    }
  }
}

Available tools

ToolArgumentsDescription
create_remindertitle (required), kind (timer|reminder), minutes, at (ISO8601), recur (none|daily|weekdays|weekly|monthly), notesCreate a reminder or timer. When kind is omitted it is inferred: at→reminder, minutes→timer.
list_remindersincludeCompleted (bool, default false)List reminders with their ids for follow-up calls.
update_reminderid (required), title, minutes, at, recur, notesPartial update — only provided fields change. Time via at (reminder only) or minutes, not both. Editing a completed item reactivates it.
complete_reminderidMark complete.
snooze_reminderid, minutes (default 10)Postpone.
delete_reminderidDelete.

Security notes

  • The server binds 127.0.0.1 only — it is unreachable from your network or the internet.
  • Only requests carrying the correct Authorization: Bearer <token> are accepted, blocking other local processes.
  • After “Reset token”, reconfigure every client with the new token.
  • If the port auto-moves, use the URL shown in Settings and reconfigure once.
  • Turn the switch off in Settings to stop listening entirely.

中文

MCP 接入指南

DueIsland · 让任意 AI 客户端管理你的提醒

DueIsland 内置本地 MCP 服务，任何支持 MCP 的 AI 客户端（Claude Code、Claude Desktop、Cursor、Codex、VS Code 等）都能创建、查询、修改、完成、延迟、删除你的提醒，改动实时反映到刘海与主窗口。

开启

  1. 打开 DueIsland ▸ 设置 ▸ MCP 服务，打开开关。
  2. 复制设置页显示的地址与鉴权头（每台设备唯一，重置 token 后旧配置失效）。

服务只监听 127.0.0.1（本机，纯 HTTP——回环流量不出本机，且公共 CA 不给 localhost 签发证书，此处 HTTPS 既不可行也无必要），并要求 Authorization: Bearer <token> 鉴权。

通用连接信息

任何客户端只需这两项：

项值
地址 URLhttp://127.0.0.1:<PORT>/mcp
鉴权头 HeaderAuthorization: Bearer <TOKEN>
传输 TransportStreamable HTTP

<PORT> 默认 51837；若被占用，服务会自动顺延到下一个可用端口，以设置页显示的地址为准。<TOKEN> 从设置页复制。请用 127.0.0.1 而非 localhost：部分客户端会把 localhost 解析成 IPv6 的 ::1，服务未绑定该地址。

可用工具

工具参数说明
create_remindertitle（必填）、kind(timer|reminder)、minutes、at(ISO8601)、recur(none|daily|weekdays|weekly|monthly)、notes建提醒/计时器。省略 kind 时按 at→reminder、minutes→timer 推断。
list_remindersincludeCompleted(bool，默认 false)列出提醒（含 id，供后续按 id 操作）。
update_reminderid（必填）、title、minutes、at、recur、notes部分更新——只改传入的字段。时间用 at（仅 reminder）或 minutes 二选一。编辑已完成条目会将其复活。
complete_reminderid标记完成。
snooze_reminderid、minutes（默认 10）延迟。
delete_reminderid删除。

安全说明

  • 服务只绑 127.0.0.1，局域网/公网不可访问。
  • 只有携带正确 Authorization: Bearer <token> 的请求被接受，挡掉同机其它进程。
  • 「重置 token」后，所有已配置的客户端需用新 token 重新配置。
  • 端口被占用自动顺延时，以设置页显示的地址为准，重新配置一次即可。
  • 不用时在设置页关闭开关即停止监听。
